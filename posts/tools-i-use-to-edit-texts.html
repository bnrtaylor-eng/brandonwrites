Tools I Use to Edit Texts

15 May 2025

A short guide to how I use Git, XML, and static sites to build collaborative editions.

My editorial work relies on a small, consistent toolkit: Git for version control, TEI-XML for encoding, and Netlify for deployment. Together, these tools let me build transparent, collaborative digital editions that are easy to maintain and simple to publish.

I use Git to track changes across each document. Every contribution—whether from students or collaborators—is versioned and documented. This makes it easy to review, revert, and credit editorial work over time.

The core editing happens in TEI, an XML standard designed for textual scholarship. I encode each letter or document using a shared schema that marks structure, names, places, and editorial choices. This allows for rich semantic data and structured interpretation while keeping the source readable.

Netlify hosts and deploys the site as a static build, so there’s no database to manage and no barrier to viewing the work. Updates are immediate, and anyone with a browser can access the materials. This setup is ideal for student-driven, open-access editorial projects like Early Modern Letters.
